# Stage Delivery Report

Use this report format after each completed stage.

## Required Sections

1. Что создано
2. Что изменено
3. Как запустить
4. Что проверить
5. Что специально не делалось
6. Следующий шаг
7. Опционально

## Rules

- Be specific and concise.
- Mention important files by path.
- If there is no `package.json`, say that local npm commands are not available yet.
- Clearly list scope that was intentionally not done.
- Recommend the next stage from `TASKS.md`.
