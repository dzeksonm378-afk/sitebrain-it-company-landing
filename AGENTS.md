# AGENTS.md

Правила для AI-разработчика в проекте **SiteBrain | IT Company Group**.

## Базовый порядок работы

1. Сначала изучать текущую структуру проекта: файлы, папки, `package.json`, конфиги, существующие компоненты и стили.
2. Не переписывать проект с нуля без явной причины.
3. Не удалять существующие файлы без необходимости.
4. Если файл уже существует, аккуратно обновлять его, сохраняя полезную структуру.
5. Перед добавлением новых решений проверять, нет ли уже похожего паттерна в проекте.

## Ограничения Stage 0 / v1

- Не добавлять лишние npm-зависимости.
- Не делать backend без отдельной задачи.
- Не добавлять базу данных без отдельной задачи.
- Не добавлять авторизацию, личные кабинеты, оплату и сложные роли без отдельной задачи.
- Не делать форму заявки с серверной отправкой без отдельной задачи.
- Не добавлять сложные анимации и тяжёлые визуальные эффекты без необходимости.

## Архитектура

- Использовать Next.js App Router.
- Использовать TypeScript.
- Использовать Tailwind CSS.
- Контент лендинга выносить в отдельные data-файлы, основной файл: `src/data/siteContent.ts`.
- `src/app/page.tsx` должен собирать секции, а не хранить весь контент и всю разметку.
- Компоненты делать маленькими, понятными и переиспользуемыми.
- Секции лендинга держать в `src/components/sections`.
- Общие UI-компоненты держать в `src/components/ui`.

## Контент

- Писать понятно для бизнеса, а не только для разработчиков.
- Объяснять пользу через заявки, клиентов, порядок в процессах, автоматизацию и экономию времени.
- Не перегружать сайт техническими словами.
- CTA должен вести к обсуждению проекта, звонку или переписке.

## Дизайн и адаптивность

- Соблюдать современный B2B IT-стиль.
- Делать обязательную адаптацию под мобильные устройства.
- Проверять, что текст не обрезается и элементы не накладываются друг на друга.
- Использовать строгие карточки, аккуратные границы, крупные заголовки и умеренные акценты.
- Не уходить в кислотный hacker-style.

## Отчёт после каждого этапа

После завершения этапа давать отчёт:

1. Что создано.
2. Что изменено.
3. Как запустить.
4. Что проверить.
5. Что специально не делалось.
6. Следующий шаг.
